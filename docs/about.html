About p5.export

One-click lossless animation capture for p5.js sketches — and any other canvas-based animation. Point it at a project folder, press Record, and a ready-to-post MP4 (or a PNG sequence) lands in [project]/export/ right next to the sources.

Why

Realtime encoders like MediaRecorder or screen-record tools visibly stumble on dense p5 animations — wave shifts, flow fields, moiré patterns. The bitrate can't keep up, the encoder drops frames, and the final video doesn't look like the sketch.

p5.export renders each frame, hands it to an offline H.264 encoder (Mediabunny + WebCodecs) or writes it as a lossless PNG, and only finalises the MP4 once every frame is in. The output shows what the sketch actually drew — not what a realtime encoder could squeeze into the bandwidth.

How it works

The sketch runs inside an iframe built from your project's real index.html, so addons that capture references at load time (p5.waves, p5.svg, p5.sound, …) behave exactly as they do standalone. A small bridge script is injected at the top of the iframe's <head> to relay each post-draw tick back to the parent, and to override millis() and deltaTime so sketch-time advances in lockstep with the captured frames — independent of wall-clock.

Because capture is offline, wall-clock time is decoupled from playback time. Lightweight sketches capture faster than real-time (10 s at 60 fps in a second or two). Heavy sketches on the master profile get 500 ms per frame to finish drawing and still produce smooth 60 fps output, as long as the animation reads time from millis() / deltaTime.

What you get

  • Instagram-ready MP4 — faststart, mp42 brand, keyframe per second, H.264 High profile. Tuned for IG feed & Reels.
  • Edit-master MP4 — 60 fps / 20 Mbps, for Premiere & DaVinci.
  • Lossless PNG sequence — streamed straight to disk via the File System Access API, so you can render masters longer than RAM would allow. Assemble with ffmpeg locally.

Requirements

Chromium-based browsers only (Chrome / Edge) — the File System Access API is used to grant readwrite permission on the project folder so exports can land next to the sources without a Downloads round-trip. An HTTPS or localhost origin is required for FSA to work.
